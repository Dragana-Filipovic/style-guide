Getting started

Feedback appreciated!

The Web Style Guide aims to support you in building awesome responsive web sites and apps. If you struggle using the assets, miss a component or face any other issue please feel free to file an issue on github or use the feedback form.

Get the Web Style Guide

Use npm

Our recommended way to integrate the Web Style Guide into a project is to use npm.

npm install @axa-ch/style-guide

Direct download

If you don't need the Web Style Guide as a versioned project dependency check out our GitHub releases page .

Use bower

We haven't published to bower, yet. But you can get the dependency using the URL to the repository.

bower install https://github.com/axa-ch/style-guide.git

What's included

css
The precompiled CSS assets for `style.less` and `normalize.less`. We recommend using the LESS assets to maintain configurability (see customization section below).
docs
A copy of the docs you're currently reading.
fonts
Here you'll find the icon font used within the web design guide in various formats.
images
Contains all image assets used by the web design guide.
jquery
Compiled (from CoffeeScript) and optionally minified jQuery plugins.
less
The raw LESS files you can use to compile your custom CSS assets.

Using the Web Style Guide

Basic template 

<!DOCTYPE html>
<html lang="en">
  <head>
    <title>AXA Web Style Guide - Basic template</title>

    <!-- meta tags as explained in the recommendations page -->
    <meta charset="utf-8">
    <meta http-equiv="X-UA-Compatible" content="IE=edge">
    <meta name="viewport" content="width=device-width, minimum-scale=1.0, maximum-scale=1.0">

    <!-- add polyfills, SEO and Social metadata tags here -->

    <!-- include the procided style.min.css or your own css assets -->
    <link href="css/style.min.css" rel="stylesheet">
  </head>
  <body>
    <h1>Hello, world!</h1>

    <!-- optional: add jQuery plugins -->
    <script src="libs/jquery/1.11.1/jquery.min.js"></script>
    <script src="libs/moment.js/2.9.0/moment.js"></script>
    <script src="js/style.all.min.js"></script>

  </body>
</html>

LESS/CSS

You can either use the CSS files provided in the packages or compile the LESS assets on your own.

Using the CSS files

We do not recommend using the precompiled CSS files in a productive environment since you can't configure anything and need to add the whole bloat of CSS even if you just need a tiny bit.

To use the precompiled CSS files just reference them from your HTML (see the tempalte above).

When using the CSS files make sure that the assets referenced from the CSS files (e.g. images at ../images ) are available.

Compiling the LESS files on your own

Compiling the LESS files is the recommended approach since you gain control over paths and what to include or not.

  • Inculde the blocks you need and omit those you don't need
  • Customize LESS variables
  • Use custom post processing

Basic LESS template

The following examples assume that you have added the less/ folder of the Web Style Guide to the LESS compilers paths.

Including needed elements only

normalize and basics are the only @import s required. Add other @import s as needed.

// required
@import 'normalize';
@import 'style/basics';

// add utils, grid and blocks as needed
@import 'style/utils';
@import 'style/grid';

@import 'style/blocks/header';
@import 'style/blocks/footer';
Including the entire Web Style Guide

This approach may be used for prototyping but it is not recommended in a productive setup since it will add a lot of CSS markup you don't need.

@import 'normalize';
@import 'style';

LESS variables

Have a look at the well commented less/variables.less file to see what variables you can override.

Post processing

Add autorefixer and pseudoelements to your build to make the css output work in older browsers.

Have a look at the postcss plugin list for further processing.

jQuery plugins

Use the jQuery plugins to give your app some interactivity. Note that these plugins are an early draft and every contribution is highly appreciated.

You can include each plugin individually or the axa-wsg.jquery.all(.min).js to get them all.
