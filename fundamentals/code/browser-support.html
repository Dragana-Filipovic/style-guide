Browser support

Customers, partners and employees are using an extensive amount of different browsers. Therefore, websites and applications should cater to all of them and not rely on proprietary technologies in or for different browsers. The W3C standards are also our standards.

Internet Explorer 8 & 9

You'll need to add some of the polyfills listed below to add support for IE 9.

With the current release Internet Explorer 8 is NOT supported. Please consider adding a "Your browser is not supported page" or add support for IE 8 on your own. The polyfills listed below will do the major parts of the job.

Version Polyfill
up to IE 9
HTML 5 Elements

Don't use HTML 5 tags or add the HTML5SHIV polyfill.
up to IE 8
Media queries

Add the respond.js polyfill to support media queries.
up to IE 8
CSS 3 pseudo element selectors (::selector)

Replace every double colon selector (::selector) with a single colon one (:selector).

We recommend to use postcss-pseudoelements postcss plugin to automate this task.
up to IE 8
Border radius

Accept the square corners or add the css3pie polyfill. Note that this polyfill may slow down the page rendering on IE 8.
up to IE 8
Background size

Add the background-size-polyfill to support this property in IE 7/8.
up to IE 8
jQuery

jQuery 2.0 dropped support for IE 8 in favour of a major performance boost. Use the 1.x Version instead to support IE 8.
-
Other

Use caniuse.com to look up browser support for a specific feature.

Take a look at the Modernizr wiki to find further polyfills.
