Button

A button indicates one or multiple actions to the user.

Table of contents
  • Primary button
  • Secondary button
  • Floating action button

Primary button

A buttons triggers an action to the page or an object. An additional icon can be added.

See code

CSS
  • Prevent whitespace at the start and the end of the button. It will add some additional spacing between the icon and the button.

  <svg class="button__icon button__icon--left icon-svg">
    <use xlink:href="/images/icons.svg#cross"></use>
  </svg>Icon and badge<span class="button__badge badge">159</span>
</div> 
<div class="button">Icon right
  <svg class="button__icon button__icon--right icon-svg">
    <use xlink:href="/images/icons.svg#cross"></use>
  </svg>
</div>

Use the CSS classes mentioned in the HTML tab as mixins.

UX Checklist

  • Icon only buttons are usually hard to understand. Do not use withouth proper testing.

Secondary button

The secondary button provides additional actions to the user. In cases where the primary call to action would be to strong, a secondary button might take his place.

  <svg class="button__icon button__icon--right icon-svg">

  </svg>
</div>

Use the CSS classes mentioned in the HTML tab as mixins.

UX Checklist

  • If the main goal of you site is information a secondary call to action can be used.

Floating action button

The floating action button consists of an icon only, it can't contain a label. This type of button is used to represent an action that is important and ubiquitous.

See code
HTML
CSS
  • Do not add any whitespace within the button since it would miss-position the icon 
<div class="button button--floating">
  <svg class="button__icon icon-svg">
    <use xlink:href="/images/icons.svg#feedback"></use>
  </svg>
</div>

Use the CSS classes mentioned in the HTML tab as mixins.

UX Checklist

  • Avoid this type of button for destructive actions like delete, nonspecific actions, alerts or errors.
