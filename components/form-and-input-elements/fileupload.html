File Upload

A fileupload element lets users upload one or multiple files to an application or into a form. Its different states are described in this document after a working example.

Table of contents
  • Example
  • Dropzone
  • File List
  • Native File Upload
  • Single File Upload

Example

This is a working example of the fileupload component. Please drag a file into the dropzone.

Dropzone

The dropzone is the initial state of every fileupload and should be used wherever possible. If a file upload is not feasible, a classic file upload should be used.

See code
HTML
JavaScript
<div class="l-container">
  <div class="form">
    <div class="form__group">
      <div class="form__group__label">Dateien</div>
      <div class="form__group__control">
        <div class="multi-upload">
          <div class="dropzone">
            <div data-dropzone="dropzone" class="dropzone__container">
              <input type="file" class="dropzone__input"/>
              <div class="dropzone__icon-container">
                <svg class="icon-svg dropzone__icon">
                  <use xlink:href="/images/icons.svg#upload"></use>
                </svg>
              </div>
              <div class="dropzone__text-container">
                <div class="dropzone__text"><span>Drag file here or</span>&nbsp;<a class="dropzone__text__upload-link">click to choose</a></div>
              </div>
            </div>
          </div>
          <div class="multi-upload__remarks">Nur Bilder (.jpg, .png, .gif) oder Dokumente (.pdf, .docx).<br/>Maximal 2 MB pro Datei</div>
        </div>
      </div>
    </div>
  </div>
</div>

Use the data-dropzone plugin to handle the dragover state of the dropzone. If you implement the behavior yourself, just add the dragover modifier to the dropzone__container block.

UX Checklist

  • Most mobile devices don’t support dragging files to an upload area. Use the native file picker as a fallback. Use the same solution for IE 9.
  • Start uploading files, as soon as the user drops them on the dropzone.
  • Make sure that all files are uploaded before the form is submitted.

UX Checklist

  • In case of an error tell the user exactly what happened. If the user needs to change something, explain as short as possible but as detailed as necessary what he needs to do.

Native File Upload

The native file upload should be used for browsers that do not support drag and drop operations for files.

This is the case for most mobile devices and ancient browser like IE 9.

See code
HTML
<div class="l-container">
  <div class="form">
    <div class="form__group">
      <div class="form__group__label">Fallback for dropzone on IE 9</div>
      <div class="form__group__control">
        <div class="multi-upload">
          <div class="dropzone-fallback">
            <input type="file" class="dropzone-fallback__input"/><a class="dropzone-fallback__button button button--primary">
              <svg class="icon-svg dropzone-fallback__button__icon">
                <use xlink:href="/images/icons.svg#upload"></use>
              </svg><span class="dropzone-fallback__button__text">Click to upload files</span></a>
          </div>
        </div>
      </div>
    </div>
  </div>
</div>

Single File Upload

Sometimes only one file should be uploaded. The uploading and the uploaded file states should replace the dropzone in this case. In case of an error, display the message below the dropzone.
